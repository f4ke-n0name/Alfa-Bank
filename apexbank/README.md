# APEX Bank — Архитектура

## Почему VIPER

Компоненты VIPER взаимодействуют друг с другом через протоколы. Это обеспечивает слабую связность позволяет легко заменять или модифицировать компоненты.

**VIPER** явно выделяет пять ролей, и каждая тестируется отдельно:

| Слой | Роль | Зависит от |
|---|---|---|
| View | Отображает ViewState, пробрасывает события | Presenter (через протокол) |
| Interactor | Бизнес-логика, работа с данными | Repository (через протокол) |
| Presenter | Связывает View и Interactor, форматирует данные | View + Interactor + Router |
| Entity | Доменные модели (BankCard, Transaction…) | — |
| Router | Навигация между модулями | UIKit только здесь |

Главное правило: **UIKit живёт только в View и Router**. Interactor и Repository не знают об экранах вообще.

## Диаграмма модуля (на примере CardDetail)

```
┌─────────────────────────────────────────────────────────┐
│                      VIPER Module                       │
│                                                         │
│  ┌──────────┐   events    ┌───────────────┐             │
│  │          │────────────▶│               │             │
│  │   View   │             │   Presenter   │             │
│  │          │◀────────────│               │             │
│  └──────────┘  ViewState  └───────┬───────┘             │
│                                   │                     │
│                         ┌─────────┴─────────┐           │
│                         │                   │           │
│                  ┌──────▼──────┐    ┌───────▼──────┐    │
│                  │ Interactor  │    │    Router    │    │
│                  └──────┬──────┘    └───────┬──────┘    │
│                         │                   │           │
└─────────────────────────┼───────────────────┼───────────┘
                          │                   │
                 ┌────────▼────────┐   открывает экраны
                 │  Repository     │
                 │  (протокол)     │
                 └────────┬────────┘
                          │
                 ┌────────▼────────┐
                 │   Domain        │
                 └─────────────────┘
```

## Навигационный флоу

```
┌────────────┐      ┌─────────┐
│  Register  │<---- │  Auth   │ ──── старт приложения
└────────────┘      └────┬────┘
                         │ session
                         ▼
                  ┌─────────────┐
                  │  CardList   │ ◀─── возврат обновляет баланс
                  └──────┬──────┘
                         │ card + session
                         ▼
                  ┌─────────────┐
                  │  CardDetail │
                  └──────┬──────┘
                         │ fromCard + session
                         ▼
                  ┌─────────────────┐
                  │  Transactions   │
                  └─────────────────┘
```

Каждый переход — это вызов `Router`. Presenter не знает, как устроен следующий экран.
